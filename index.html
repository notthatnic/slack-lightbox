The challenge

The description of the challenge, as I receieved it, is as follows:

This exercise is meant to demonstrate:

  • The ability to access to a public API and successfully retrieve data from it;
  • The ability to display that data on a page using only native JavaScript (no libraries); and
  • The ability to update the UI of a page without refreshing the page.

I'd like you to create a web page that shows a photo in a lightbox view, with the ability to move to the next / previous photos and display the photo title.

Cool.

My approach

I like having a lot of visibility into the feature development lifecycle. I can be a better engineer when I understand the fundamental goals of a feature, if for no other reason than it makes me better able to evaluate if the proposed implementation the right one (for the user, for the codebase, etc.).

To that end, I decided to create some context for the development of a feature that would require a lightbox. I used myself and my weirdo friends as a stand in for the user population of Slack because a) we actually use the product regularly, b) I have insight into our needs as users, and c) it’s my challenge and I can do what I want.

So here’s the use case:

Nicole et al have developed a very visual conversational style. As lovers of reaction gifs and infographics, Slack channels are full of each.

These users are HipChat refugees, a land where there was no giphy integration, and as such most have curated libraries of reaction gifs from across the web. These images are often shared as links (as opposed to uploading files).

Users have expressed a desire for an easy way to browse images shared to a channel.

A note about the CSS

I decided to use an atomic/functional approach to the styles, because:

  • it helps enforce design consistency
  • it's easier to make small changes to visual components without fighting inheritance and specificity
  • it can help keep overall stylesheet file size down

I've opened with the benefits of this approach because anyone seeing the proliferation of classes that come with functional CSS for the first time is at high risk of table-flipping and hair-rending. I’m not indifferent to the fact that the markup starts to look a little nutso, but the performance benefits of smaller stylesheets vastly outweigh any hits taken from verbose HTML.
